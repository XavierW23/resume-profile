# Xavier White GitHub Portfolio Plan

This folder contains public-safe GitHub portfolio assets for Xavier White's engineering brand:

**Xavier White - Software Engineer specializing in Autonomous AI Systems**

## Public Portfolio Strategy

Agency OS should remain private because it is active business IP. These public examples demonstrate the engineering patterns around the system without exposing proprietary workflows, prompts, client logic, or business data.

This repo is the single public profile package. The examples below live as folders inside `portfolio-examples/` instead of separate repositories.

## Recommended Portfolio Folders

1. `portfolio-examples/ai-workflow-examples`
   - Structured AI outputs
   - Validation gates
   - Prompt/output contracts
   - Learning-event examples

2. `portfolio-examples/runtime-supervisor`
   - Heartbeat monitoring
   - JSON runtime state
   - Recovery logic
   - Operational logging

3. `portfolio-examples/browser-automation`
   - Playwright/Chrome automation
   - Evidence capture
   - UI state inspection
   - Safety checks

4. `portfolio-examples/github-actions-automation`
   - Scheduled jobs
   - Manual workflow dispatch
   - State refresh
   - Notification-ready automation

5. `portfolio-examples/engineering-utilities`
   - Logging
   - Retries
   - Config loading
   - JSON artifact helpers

## Publishing Checklist

- Keep this repository public.
- Keep the examples organized as folders under `portfolio-examples/`.
- Keep Agency OS private.
- Do not include real customer data, production prompts, credentials, API keys, Make webhook URLs, Slack tokens, or proprietary business rules.
- Add screenshots only if they are sanitized and do not reveal client or business-sensitive data.
